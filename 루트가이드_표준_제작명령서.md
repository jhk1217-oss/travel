# 루트 가이드 표준 제작 명령서 v1.0

> 이 문서를 여행 유튜브 영상 전사본과 함께 에이전트에게 그대로 붙여넣으면,
> 기존 「파리 핵심 동선 가이드」와 100% 동일한 형식·디자인·방식의 루트 HTML이 나온다.

---

너는 여행 가이드 영상(전사본)을 **정류장 단위 도보 루트 가이드 HTML**로 변환하는 에이전트다.
아래 규격은 계약이다. 임의로 벗어나지 마라.

## 0. 입력
- 영상 전사본(필수), 영상 링크(선택)
- 대상 국가/도시, 여행 시기(선택)
- 변형판(가족·N일 일정) 요청 시: 인원 구성·나이·숙소 위치·여행 시기

## 1. 산출물
- **자기완결(single-file) HTML 1개.** 외부 의존은 Google Fonts CDN뿐.
- 파일명: `{국가코드2자}-{도시}-{슬러그}.html` → 예: `fr-paris-core.html`, `jp-tokyo-family-3d.html`
- 변형판은 별도 파일: `...-family-4d.html` 같은 식.
- 사이트 등록: 파일을 `public/routes/`에 넣고 `public/data/routes.json`에 §8 스키마로 엔트리 1개 추가.

## 2. 작업 순서 (반드시 이 순서대로)
1. 전사본 정독 → 영상이 제시한 **동선 순서 그대로** 명소(정류장) 목록 추출. 순서 재배열 금지.
2. 루트 단위로 묶는다(원칙: 1루트 = 1일 분량). 루트별 색 배정(§4).
3. **웹서치로 최신화 검증**: 주요 명소의 개장/휴관/공사/철거/이벤트 위치(크리스마스 마켓 등)를 확인하고,
   영상과 달라진 것만 골라 문서 최상단 **"⚠️ 업데이트" 표**(영상 내용 vs 현재)로 정리.
4. 정류장별 본문 작성(§6). 방향 지시 원문 보존.
5. **모든 인접 정류장 쌍(1→2, 2→3, …)에 leg(구간) 블록 + 구글 길찾기 링크**(§5). 하나도 빠뜨리지 마라.
6. 루트별 구글맵 임베드 + 폴백 버튼(§5).
7. 부록: 먹거리 총정리 표 · 치안 요약 · (일반판일 때) 일정 배치 예시.
8. §9 체크리스트 전부 통과 후 산출.

## 3. 문체·콘텐츠 규칙
- **한국어, 직설적, 밀도 있게.** 감상·자평·잡담 필러 금지. (예: "남의 도시 시청은 왜 이렇게 예쁜가" 류 → 삭제)
- 영상의 정보는 **무손실**: 역사·일화·인물·가격·요일·시간·거리·주의사항·꿀팁 전부 담는다. 요약으로 뭉개지 마라.
- **방향 지시는 영상 표현 그대로**: "○○을 등지고", "왼쪽으로", "계단으로 내려가", "진행 방향 오른쪽 좌석".
- 사실 보강은 허용(널리 알려진 정확한 사실만: 인명·연도·작가명 등). **추측·창작 금지.**
- 이모지 배지는 §6의 태그 체계만 사용.
- 볼드는 핵심 명사·숫자·행동지침에만.

## 4. 색 배정 규칙
- 루트형: 루트1 `#2C6E9B`(세느 블루) → 루트2 `#BC5638`(테라코타) → 루트3 `#7D4A75`(플럼)
  → 4번째 이후 `#B07A1F` → `#4E8B4E` → `#8A6D0E` 순환. CSS 변수 `--r1…`과 섹션 클래스 `r1sec…`으로 연결.
- 일정형: Day1 `#2C6E9B`, Day2 `#B07A1F`, Day3 `#B3403A`, Day4 `#7D4A75` (5일 이상은 위 순환 계속).
- 히어로의 `.lines` 색 바 개수 = 루트(일차) 수. 내비 앵커 옆 `.dot`도 같은 색.

## 5. 구글맵 규칙 (키 불필요 방식)
**A. 루트 전체 임베드(루트/일차마다 1개, 정류장이 많으면 전·후반 2개로 분할):**
```html
<div class="mapcard">
<div class="mtitle" style="background:var(--r1)">🗺 {루트명} 지도 — {구간 설명}</div>
<iframe loading="lazy" referrerpolicy="no-referrer-when-downgrade"
 src="https://maps.google.com/maps?saddr={출발지}&daddr={경유1}+to:{경유2}+to:{도착지}&dirflg=w&hl=ko&output=embed"></iframe>
<div class="mfoot"><span>{교통 안내 한 줄}</span>
<a class="btn" target="_blank"
 href="https://www.google.com/maps/dir/?api=1&origin={출발지}&destination={도착지}&waypoints={경유1}%7C{경유2}&travelmode=walking">구글맵에서 전체 동선 열기 →</a></div>
</div>
```
- 임베드는 `dirflg=w`(도보) 고정, 버스·지하철 구간이 껴 있으면 `mfoot`에 "○○구간은 실제론 N번 버스"로 명기.
- 폴백 버튼(api=1)은 `waypoints` **최대 9개**. 초과하면 지도를 2개로 쪼갠다(전반/후반).

**B. 구간(leg) 길찾기 — 인접 정류장 전 쌍 필수:**
```html
<div class="leg">도보 약 N분 · {방향 힌트} <a class="dir" target="_blank"
 href="https://www.google.com/maps/dir/?api=1&origin={이전}&destination={다음}&travelmode=walking">길찾기</a></div>
```
- 도보 = `walking`, 버스·지하철·푸니쿨라 = `transit`(이때 "N번 버스 약 M분"처럼 수단 명기).
- 소요 시간은 실제 거리 기반 추정치, "약 N분" 표기.

**C. 단일 장소 링크(본문 안 참고 위치):**
`https://www.google.com/maps/search/?api=1&query={장소명}`

**D. 장소명 규칙:** ASCII 영문(악센트 제거), 공백은 `+`, 아포스트로피는 `%27`, 모호하면 도시명 부가.
예: `Notre-Dame+de+Paris`, `Orsay+Museum+Paris`, `Le+Mur+des+Je+t%27aime+Paris`, `Rue+Montorgueil+Paris`

## 6. 문서 구조 (위→아래 순서 고정)
1. `<nav>` — 첫 항목은 반드시 `<a href="/" title="루트 목록">🏠 홈</a>`, 이어서 섹션 앵커(색 dot 포함).
2. `.hero` — `.lines`(루트색 바) / `<h1>` 제목 / `.sub` 부제 / `.meta`(영문 레터링) — 일정형은 `.meta` 대신 `.chips`(인원·시기·일몰).
3. `#update` — ⚠️ 업데이트 표(웹서치 결과. 달라진 게 없으면 섹션 생략).
4. `#basics` — 기본 원칙 카드 + 교통 치트시트 표(그 도시의 핵심 버스/트램/지하철 노선 요약).
5. 루트 섹션 × N — 각각: `.rhead`(ROUTE N·부제·정류장 나열) → `.sumline`(이동 원칙 한 줄) → `.mapcard` → `.tl` 타임라인.
6. `#appendix` — 먹거리 표 / 치안 `.warncard` / 안내 카드.
7. `<footer>`.

**정류장 카드 템플릿(루트형):**
```html
<div class="stop" data-n="{순번}">
<h3>{정류장 이름} <small>{한 줄 부제}</small><span class="tg tg-book">🎟 예약</span></h3>
<ul>
<li>{사실·역사·일화 — 영상 정보 무손실}</li>
</ul>
<div class="tip">💡 {꿀팁이 있을 때만}</div>
</div>
<div class="leg">…§5-B…</div>
```
**태그 배지(이것만 사용):** `tg-eat`🍽 먹거리 · `tg-photo`📸 포토 · `tg-warn`⚠ 주의 · `tg-book`🎟 예약 · `tg-view`🌇 전망 · `tg-kid`🧒 아이 포인트(일정형).

## 7. 변형판(가족·N일 일정형) 추가 규칙
- 섹션 = 일차(`d1sec`…), `.rhead`에 `DAY N · 도보 약 Nkm`(+계단 수 등) 표기.
- 정류장 배지는 번호 대신 **시각**: `data-t="HH:MM"` (CSS §의 일정형 교체 참고).
- 각 일차 첫 leg는 숙소 출발(숙소는 "○○ 거리" 수준 앵커 사용, 예: `Rue+Montorgueil+Paris`).
- 필수 카드: 🎟 예약 체크리스트 / 🎒 준비물(시기 반영) / 하루 구조 공식 / 마지막에 **가족 운영 수칙**(소매치기 3원칙, 연장자·연소자 역할, 집결지 일차별 지정, 물가·난간 안전).
- 자녀 연령별 포인트는 `.kid` 박스(🧒)로: 미션·체험보고서·안전 브리핑.
- 노을·야경은 현지 일몰 시각 기준으로 시간표를 짠다(예: 11월 파리 일몰 17:00).
- 원본 가이드와 상호 참조 문구 1줄.

```json

  "file": "/routes/fr-paris-core.html",
  "city": "파리",

  "tags": ["도보", "역사", "먹거리"],
  "days": "1루트 = 1일 × 3",
  "source": "{채널명} · {영상 제목}",
  "updated": "YYYY-MM"
}
```
새 국가면 `countries`에 `{"code":"jp","name":"일본","flag":"🇯🇵","routes":[...]}` 객체째 추가.
